Prime Contractor

Multi-agent code generation that routes tasks to the cheapest model that can handle them, merges each result immediately, and gives you full cost and quality telemetry.

What It Does

The Prime Contractor takes a batch of code generation tasks (derived from a plan + requirements) and processes them one at a time through a tiered agent hierarchy. Each feature is generated, merged, tested, and committed before the next one starts.

Cost-Optimized Tier Routing

Every task is classified by 12 complexity signals (blast radius, dynamic dispatch, inheritance depth, cross-file edges, etc.) and routed to the cheapest model tier that can handle it. No task pays for Opus if Haiku will do.

Tier Generator Cost When Used
TRIVIAL Template Registry $0 Zero Boilerplate, __init__.py, config stubs
SIMPLE Ollama (local 7B) local ~$0.00 Single-function, no cross-file deps, low blast radius
MODERATE Ollama file-whole local ~$0.00 Multi-element files, class hierarchies, moderate deps
COMPLEX Primary Contractor cloud ~$0.50 Cross-file, dynamic dispatch, deep inheritance, high blast radius

Agent Architecture

Four agent roles with typed contracts at every handoff boundary. The orchestrator is stateless — all coordination state lives in the task queue and context dict, making runs reproducible and resumable.

MicroPrime Engine local
SIMPLE + MODERATE tiers — Ollama (startd8-coder 7B)
  • Element-level generation with AST splicer (Python)
  • File-whole generation for MODERATE tasks
  • Decomposition fallback: Class & Function strategies
  • Circuit breaker: 8 per-file, 12 per-run
  • Auto-escalates to cloud on repeated failure
Primary Contractor cloud
COMPLEX tier — Sonnet (lead) + Haiku (drafter)
  • Lead agent writes 8-section spec (budget-capped at 4K tokens)
  • Drafter agent implements from spec (4 draft modes)
  • Lead agent reviews, scores 0–100
  • Up to 3 iterations if score < 80
  • Expensive model reviews; cheap model generates
Repair Pipeline
Post-generation fixup — deterministic, no LLM
  • fence_strip: remove markdown fences
  • indent_normalize: fix indentation errors
  • import_completion: add missing imports
  • bare_statement_wrap: fix expressions-as-statements
  • Per-feature circuit breaker reset
Integration Engine
Merge, test, commit/rollback — deterministic
  • Snapshot originals before merge
  • Splice bodies or overwrite (per merge strategy)
  • Checkpoint: run tests + lint
  • Auto-commit on pass, restore snapshot on fail
  • Semantic checks: dupe defs, phantom imports, bare except

Agent-to-Agent Contracts (Keiyaku)

Every agent boundary uses a frozen dataclass — not prose, not bare dicts. When an Ollama model fails and escalates to cloud, the handoff carries structured failure context so the cloud model knows what was already tried.

K-10: ClassificationResult
Classifier → Router/Decomposer. Carries the tier label plus all 12 complexity signals. Decomposer uses signals to choose strategy, not just the tier.
K-6: EscalationHandoff
Local model → Cloud agent. Includes element FQN, attempt count, failure category, repair outcome. Renders as both JSON and human-readable summary.
K-9: EscalationRepairOutcome
Repair pipeline → Escalation context. Per-step results (which repairs were tried), AST validity before/after, final verdict. Prevents cloud from repeating failed repairs.
K-7: SemanticVerificationResult
LLM semantic validator → Engine. Verdict (pass/fail/inconclusive), confidence score, frozen issue list. Fail-open on format parse, fail-closed on content.

Safety & Resilience

The pipeline is designed so that a single bad task cannot take down the entire run, and every destructive action is reversible until explicitly committed.

🛡

Snapshot-Before-Merge

Every file is backed up before modification. Failed integrations restore the snapshot automatically.

⚡

Circuit Breakers

Per-file (8 failures) resets between files. Per-run (12) persists — detects systemic issues like Ollama being down.

🔄

Error-Informed Retry

Failed feature errors are injected into the regeneration prompt as structured feedback, so the next attempt avoids the same mistake.

📊

Checkpoint Validation

Tests + lint run after every merge. Nothing is committed unless the checkpoint passes.

🌍

Language Detection

Python stubs in Go files are caught by non-Python guard. 4 language profiles: Python, Go, Node.js, Java.

📝

Kaizen Postmortem

Every run produces disk quality scores, assembly deltas, cross-feature patterns, and kaizen suggestions for the next run.

Observability

Every agent call, cost event, and quality score flows through OpenTelemetry into the local Grafana stack. Post-mortem runs automatically after each batch.

What This Means for Engineering Managers

The Prime Contractor changes the economics of code generation by routing 80%+ of tasks to free local models, while preserving cloud quality for the tasks that need it.

Cost Control
Pay only for complexity, not volume
  • TRIVIAL/SIMPLE/MODERATE tasks cost $0 (local Ollama)
  • Only COMPLEX tasks hit cloud APIs (~$0.50 each)
  • Budget enforcement caps prompt sizes (P0-P3 priority trimming)
  • Per-run, per-feature, per-agent cost tracking with OTel export
Quality Visibility
Measure what matters, automatically
  • Dual scoring: requirement fulfillment vs. disk quality
  • Assembly delta detects systematic degradation
  • Kaizen trends track improvement across 20+ runs
  • Correlation analysis links prompt features to outcomes
Operational Safety
Nothing ships without passing tests
  • Snapshot-restore on every merge (reversible by default)
  • Circuit breakers prevent cascade failures
  • Typed A2A contracts prevent silent data corruption
  • Non-Python language guard prevents wrong-language stubs
Team Leverage
Automate the routine, escalate the hard
  • 96-task batches processed with 3-4 runs, not 96 PRs
  • Postmortem reports surface patterns humans should review
  • Kaizen suggestions are actionable (not just metrics)
  • Resumable: crash mid-batch, pick up where you left off
