## Purpose

Review the independent oracle-calibration implementation on branch
codex/independent-oracle-fixtures. This branch replaces a measured Codex
cohort suite with canonical fixtures authored from the pricing contract.

The implementation commits are:

- db25b432 — independent fixture suite, mutant recalibration, and gate checks
- 1a2312d9 — immutable provenance and corrected re-review instructions

Do not treat a passing test suite as sufficient. The review must establish
that the scoring instrument is independent of every measured authoring cohort.

## Reviewer independence

- Codex and Gemini are the designated non-Claude review pair for this audit.
  Their disclosed cohort affiliations are acceptable because the review policy
  excludes Claude and Anthropic participation.
- A reviewer may also have an implementation role when that overlap is
  disclosed in the sign-off and provenance record.
- Blinding is preferred where practical, but an unblinded Codex/Gemini review
  pair satisfies this audit's stated acceptance policy.
- Do not use authoring-run files, vendor-generated test suites, or their
  expected outputs to validate the oracle.
- If a conflict, cohort affiliation, or tool assistance exists, record it in
  the sign-off rationale.

Then run each mutant:

Expected mutant result: every command has at least one assertion failure. An
import error, syntax error, or test harness failure is not a valid kill.

canonical fixture source and a reference test count of 38.

## Review the admission controls

Review scripts/validate_cross_tool_oracle_gate.py and its unit tests. Confirm
the gate rejects:

- a provenance record without an immutable implementation commit;
- a provenance record without independent non-Claude review evidence;
- calibration code that refers to authoring-run artifacts;
- incomplete or non-accepting reviewer sign-offs.

Run:

~~~bash
PYTHONPATH=src pytest -q -p no:cacheprovider tests/unit/test_validate_cross_tool_oracle_gate.py
python3 scripts/validate_cross_tool_oracle_gate.py
~~~

The first command should pass. The second should remain blocked until both
reviewers accept; this is the expected fail-closed state.

## Record the review

Each reviewer must update their existing entry in
oracle/reviewer-signoffs.json. Use decision accept only if all criteria above
pass. Include the reviewed commit IDs, commands run, blinding status, and any
affiliation in evidence_reviewed and rationale.

Each accepting reviewer must also add a record to
oracle-provenance.json under independent_non_claude_review:

~~~json
{

  "decision": "accept",
  "rationale": "Independent fixture, calibration, provenance, and gate review passed.",
  "date": "YYYY-MM-DD"

~~~

If either reviewer blocks the work, record the exact failure and leave both
top-level artifact statuses pending.

Only after two accepting reviews:

1. Set oracle-provenance.json status to accepted.
2. Set reviewer-signoffs.json status to accepted.
3. Run the derived gate update:

   ~~~bash
   python3 scripts/validate_cross_tool_oracle_gate.py --sync-status
   ~~~

The resulting validation-gate.json must be accepted with zero errors before
authoring or S4-S7 analysis resumes.
