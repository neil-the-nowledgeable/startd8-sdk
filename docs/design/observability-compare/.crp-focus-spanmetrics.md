# CRP Focus — Span-Metrics SLI Binding (#307, v0.3.1)

**Least-reviewed artifact:** this entire requirements doc — brand new, zero prior external review.

## Weight the review on these concerns

1. **FR-3 descriptor-profile reconciliation (highest priority).** The SDK already has a
   `span-metrics-connector` MetricDescriptor with **OTel-Collector** names (`calls_total`,
   `duration_milliseconds_bucket`, unit ms) — but the issue/acceptance cite **Tempo metrics-generator**
   names (`traces_spanmetrics_latency_seconds_bucket`, seconds). Is adding a *second* `tempo-spanmetrics`
   profile right, or should one profile be parameterized? How is the correct profile SELECTED per service
   (author-declared? surface-derived?)? Does the unit difference (ms vs s) leak into the SLO target?

2. **FR-1 `spanmetrics` surface dual-nature.** It is added to `NON_EMITTING_CONVENTION_SURFACES` (base RED
   suppresses) AND triggers span-binding. Is that dual role coherent with the existing #274 suppression +
   #285 ServiceMonitor gates (`NON_SCRAPEABLE_SURFACES`)? Should `spanmetrics` be scrapeable? Any gate that
   would now mis-fire for a `spanmetrics` service.

3. **FR-4 precedence + FR-2 parse.** Does "declared span-signal > suppress > convention" interact correctly
   with the existing `_declared_covered_kinds` suppression seam and the #300 base/functional binders (three
   declared lanes now: base, functional, span)? De-dup / double-emit risk across lanes for the same
   `(service, kind)`. Is the `DeclaredSpanSignal` shape (`attributes`, `error_selector`, `target`) complete?

4. **FR-4 availability from span-metrics (OQ-3).** errors/calls ratio needs an `error_selector` on
   `calls_total` (`status_code`/`span.status`) — is the #286-v2 availability discipline transferable, and
   what is the real Tempo error dimension?

5. **FR-7 compare.py + FR-6 accounting.** A THIRD `bound_declared_*` key + a THIRD lane — is the
   compare.py/report/dashboard surface scaling cleanly, or does this argue for a unified declared-binding
   accounting shape?
