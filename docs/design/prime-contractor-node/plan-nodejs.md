## Overview

Implement the two Node.js microservices from Google's Online Boutique microservices demo. This is a validation exercise for the capability delivery pipeline: the reference implementation provides an unambiguous target for structural comparison.

**Objectives:** Implement the two Node.js microservices from Google's Online Boutique microservices demo as structurally equivalent, diffable reproductions of the reference implementation.

### Goals

- Produce two functional Node.js microservices: currencyservice, paymentservice
- Each gRPC service implements its proto-defined contract with health checking
- gRPC services include OpenTelemetry instrumentation (where present in reference)
- Cloud Profiler code present and active (matching Node.js reference)
- Both services containerized with multi-stage Dockerfiles using `node:20.20.0-alpine` builder and `alpine:3.23.3` runtime
- Generated code is structurally comparable to the reference implementation

### Non-Goals

- Kubernetes manifests, Helm charts, Kustomize overlays (infrastructure-as-code)
- Services in other languages (Python, Go, C#, Java)
- CI/CD pipelines
- Proto files (`proto/` directories) — copied, not generated
- `package-lock.json` files — generated by `npm install`
- `client.js` (currencyservice legacy test client using deprecated `grpc` package)
- `genproto.sh` scripts (build tools)
- `.dockerignore` and `.gitignore` files

### Delivery Mechanism

Every feature produces **valid Node.js files** written to the target project root. Success is measured by structural equivalence to the reference implementation: same files, same interfaces, same patterns.

**Description:**
A single-file gRPC server implementing the `CurrencyService` proto contract. `GetSupportedCurrencies` returns all 34 currency codes from a JSON data file. `Convert` performs two-step conversion via EUR pivot currency. Includes health checking, OTel tracing (conditional), Cloud Profiler (conditional), and Pino structured logging.

**Implementation contract:**

`server.js` (198 lines):
- No `'use strict'` directive
- Pino logger setup (inline, not module):
  - `name: 'currencyservice-server'`
  - `messageKey: 'message'`
  - `formatters: { level(logLevelString, logLevelNum) { return { severity: logLevelString } } }`
- Cloud Profiler block:
  - `if(process.env.DISABLE_PROFILER)` → logs disabled
  - `else` → logs enabled, `require('@google-cloud/profiler').start({serviceContext: {service: 'currencyservice', version: '1.0.0'}})`
- gRPC instrumentation (ALWAYS registered, before tracing check):
  - `const { GrpcInstrumentation } = require('@opentelemetry/instrumentation-grpc')`
  - `const { registerInstrumentations } = require('@opentelemetry/instrumentation')`
  - `registerInstrumentations({ instrumentations: [new GrpcInstrumentation()] })`
- OTel tracing (conditional on `ENABLE_TRACING == "1"`):
  - Imports `resourceFromAttributes`, `ATTR_SERVICE_NAME`, `opentelemetry` (sdk-node), `OTLPTraceExporter`
  - `collectorUrl = process.env.COLLECTOR_SERVICE_ADDR`
  - `new OTLPTraceExporter({url: collectorUrl})`
  - `new opentelemetry.NodeSDK({resource: resourceFromAttributes({[ATTR_SERVICE_NAME]: process.env.OTEL_SERVICE_NAME || 'currencyservice'}), traceExporter})`
  - `sdk.start()`
  - Else branch: logs `"Tracing disabled."`
- Module-level requires (after OTel setup):
  - `const path = require('path')`
  - `const grpc = require('@grpc/grpc-js')`
  - `const protoLoader = require('@grpc/proto-loader')`
- Proto paths:
  - `MAIN_PROTO_PATH = path.join(__dirname, './proto/demo.proto')`
  - `HEALTH_PROTO_PATH = path.join(__dirname, './proto/grpc/health/v1/health.proto')`
- `PORT = process.env.PORT`
- Proto loading:
  - `shopProto = _loadProto(MAIN_PROTO_PATH).hipstershop`
  - `healthProto = _loadProto(HEALTH_PROTO_PATH).grpc.health.v1`
- `function _loadProto(path)`:
  - `protoLoader.loadSync(path, {keepCase: true, longs: String, enums: String, defaults: true, oneofs: true})`
  - `grpc.loadPackageDefinition(packageDefinition)`
- `function _getCurrencyData(callback)`:
  - `const data = require('./data/currency_conversion.json')`
  - `callback(data)`
- `function _carry(amount)`:
  - `fractionSize = Math.pow(10, 9)`
  - `amount.nanos += (amount.units % 1) * fractionSize`
  - `amount.units = Math.floor(amount.units) + Math.floor(amount.nanos / fractionSize)`
  - `amount.nanos = amount.nanos % fractionSize`
  - Returns `amount`
- `function getSupportedCurrencies(call, callback)`:
  - Logs `'Getting supported currencies...'`
  - `_getCurrencyData((data) => { callback(null, {currency_codes: Object.keys(data)}) })`
- `function convert(call, callback)`:
  - Try block:
    - `_getCurrencyData((data) => { ... })`
    - Gets `request = call.request`, `from = request.from`
    - EUR conversion: `euros = _carry({units: from.units / data[from.currency_code], nanos: from.nanos / data[from.currency_code]})`
    - `euros.nanos = Math.round(euros.nanos)`
    - Target conversion: `result = _carry({units: euros.units * data[request.to_code], nanos: euros.nanos * data[request.to_code]})`
    - `result.units = Math.floor(result.units)`, `result.nanos = Math.floor(result.nanos)`
    - `result.currency_code = request.to_code`
    - Logs `'conversion request successful'`
    - `callback(null, result)`
  - Catch: logs error, `callback(err.message)`
- `function check(call, callback)`:
  - `callback(null, { status: 'SERVING' })`
- `function main()`:
  - Logs `Starting gRPC server on port ${PORT}...`
  - `server = new grpc.Server()`
  - `server.addService(shopProto.CurrencyService.service, {getSupportedCurrencies, convert})`
  - `server.addService(healthProto.Health.service, {check})`
  - `server.bindAsync('[::]:${PORT}', grpc.ServerCredentials.createInsecure(), function() { ... server.start() })`
  - Logs `CurrencyService gRPC server started on port ${PORT}` inside callback
- `main()` called at module level

### F-005: Credit Card Charge Logic

**Satisfies:** REQ-NMS-008, REQ-NMS-009, REQ-NMS-012

**Output files:**
- `src/paymentservice/charge.js`

**Depends on:** F-004

**Description:**
Credit card validation and mock charge processing. Validates card number (Luhn), card type (VISA/Mastercard only), and expiration date. Returns a UUID transaction ID on success.

**Implementation contract:**

`charge.js` (86 lines):
- Imports:
  - `const cardValidator = require('simple-card-validator')`
  - `const { v4: uuidv4 } = require('uuid')`
  - `const pino = require('pino')` — creates own inline logger (separate from logger.js)
- Logger config (inline):
  - `name: 'paymentservice-charge'`
  - Same `messageKey` and `formatters` pattern as all other loggers
- Error class hierarchy (all module-local, not exported):
  - `class CreditCardError extends Error`:
    - `constructor(message)`: calls `super(message)`, sets `this.code = 400`
  - `class InvalidCreditCard extends CreditCardError`:
    - `constructor(cardType)`: calls `super('Credit card info is invalid')`
    - Note: `cardType` param exists but is unused in the message
  - `class UnacceptedCreditCard extends CreditCardError`:
    - `constructor(cardType)`: calls `super('Sorry, we cannot process ${cardType} credit cards. Only VISA or MasterCard is accepted.')`
  - `class ExpiredCreditCard extends CreditCardError`:
    - `constructor(number, month, year)`: calls `super('Your credit card (ending ${number.substr(-4)}) expired on ${month}/${year}')`
- `module.exports = function charge(request)`:
  - Destructures: `const { amount, credit_card: creditCard } = request`
  - `cardNumber = creditCard.credit_card_number`
  - Step 1 — Luhn: `cardInfo = cardValidator(cardNumber)`, `{card_type: cardType, valid} = cardInfo.getCardDetails()`
  - If `!valid` → `throw new InvalidCreditCard()`
  - Step 2 — Type check: if `!(cardType === 'visa' || cardType === 'mastercard')` → `throw new UnacceptedCreditCard(cardType)`
  - Step 3 — Expiration: `currentMonth = new Date().getMonth() + 1`, `currentYear = new Date().getFullYear()`
  - Destructures: `{credit_card_expiration_year: year, credit_card_expiration_month: month} = creditCard`
  - If `(currentYear * 12 + currentMonth) > (year * 12 + month)` → `throw new ExpiredCreditCard(cardNumber.replace('-', ''), month, year)`
  - Logs transaction: `Transaction processed: ${cardType} ending ${cardNumber.substr(-4)} Amount: ${amount.currency_code}${amount.units}.${amount.nanos}`
  - Returns `{ transaction_id: uuidv4() }`
- CVV is NOT validated (intentional — demo only)
