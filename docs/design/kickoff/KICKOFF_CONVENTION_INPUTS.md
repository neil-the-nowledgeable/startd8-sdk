## 1. Scope

The conventions and vocabulary generation must *follow* instead of *invent*: framework choice,
ORM/data layer, module-path organization, naming, domain terms. **The RUN-028 class:** a
micro-prime-routed run invented Flask-vs-FastAPI, `app.models`-vs-`app.tables`,
SQLAlchemy-vs-SQLModel — exactly what a user could have declared up front.

Planning established the two halves of the gap, and both are required (collection without reach
is worthless):

1. **No per-run declaration input exists.** `.contextcore.yaml` `guidance`
   (constraints `C-*` / preferences `PREF-*` with `appliesTo`) is operator-level;
   onboarding-metadata is **not injected into prompts** (preflight/provenance only — verified at
   `plan_ingestion_workflow.py:1871–1975`).
2. **Derived authority doesn't reach all tiers.** `project_knowledge` injects into the
   prime_contractor lead/drafter path only (`prime_contractor.py:4543–4601`); `micro_prime/` has
   **zero** `project_knowledge` references; RUN-038 showed spec-authoring and test-gen are also
   unreached. **Precision (CRP R2 code finding):** micro_prime is *not* convention-blind — it
   already injects a `convention_guidance` block (`micro_prime/context.py:86–93`, FR-CAR-5) via
   `repair/convention.py:render_convention_guidance()`. But that block is a **hardcoded
   Python-only house style** (FastAPI/SQLModel/Jinja2 from `CANONICAL_LAYOUT`), not a user
   declaration. So the bypass is **partially closed for Python idioms** and **fully open for the
   declared-manifest path** — the half FR-H1/FR-H2 require.

## 3. Requirements (Group H detail)

- **FR-H1 — Convention declaration: ingestion-generated, human-validated** *(reworded per the
  2026-06-05 operator decision, Q1)*. The convention manifest (framework, ORM/data layer, module
  paths, naming — §2.4 sketch, plus the language/stack block for FR-I2) is **generated by the
  plan ingestion workflow** as a tier-G draft derived from `plan.md`/`requirements.md` (+
  architecture docs where present), persisted as a reviewable file, and **validated/enhanced by
  the Architect** at their gate (HITM §3.3). On validation its provenance flips
  `templated/inferred` → `authored`, and the validated file is **reused across subsequent runs**
  as the authoritative declaration (the generated → validated → reused ratchet). **Greenfield:**
  ingestion derives it from the plan/requirements docs — no separate hand-seeding step;
  **brownfield:** FR-H5 precedence applies against derived evidence.
- **FR-H2 — Injection reach (the RUN-028/RUN-038 closure).** Collected convention + field-set
  authority MUST reach **every generation path that emits convention-sensitive code**:
  - **(a) lead/drafter spec authoring** — authority binds into the *spec*, not only the draft
    (RUN-038: module-source authority originates in the spec and survives review);
  - **(b) micro_prime prompts** — closing the structural bypass (zero `project_knowledge` refs
    today);
  - **(c) the test-gen path** (RUN-036 #1 / RUN-038).
  **Routing precondition, not best-effort:** until a tier can receive convention authority, the
  router MUST NOT route convention-strict work to it (RUN-028 postmortem A1/A3;
  deterministic-first review D3).
  **"Convention-strict work" (definition — the precondition's classification rule):** a task is
  convention-strict iff it emits new modules/files, new imports, framework- or ORM-touching code,
  or routes/handlers in the declared stack. A seed-task label (`convention_strict: true|false`,
  set at ingestion) **overrides** the rule; **unlabeled default = strict** (conservative — the
  RUN-028 failure mode is worse than a re-route). The keep/re-route decision must be
  deterministic and unit-testable against a labeled fixture set.
  **Existing micro_prime injection is NOT FR-H2 satisfaction:** the `convention_guidance` block
  (FR-CAR-5; `render_convention_guidance()`) is hardcoded Python house style — FR-H2's remaining
  requirement is wiring the FR-H1 *user-declared* manifest into that same injection path. A
  user-declared non-default framework (e.g. `framework: django`) MUST appear in the micro-prime
  prompt — not the hardcoded FastAPI text.
  **Injection priority within the prompt budget (resolves former OQ-1's mechanism half):** the
  FR-H1 declared-convention block rides the existing authority cap
  (`micro_prime/engine.py:428–431`) at **highest priority** — above field-set/`upstream_interfaces`
  blocks and above the static `convention_guidance` text — so a user-declared framework is never
  truncated while a hardcoded default survives.
- **FR-H3 — Onboarding-metadata as declared input.** Its provenance entry records *which consumed
  fields drove what* (the §2.2 consumption map), extending FR-X4 provenance to the ingestion
  layer — so reviewers can see what the user-provided context actually influenced.
- **FR-H4 — Corpus alignment (advisory).** When the controlled corpus ships, declared convention
  manifests become corpus *seed inputs* and the corpus becomes the durable layer feeding the
  per-run `ProjectKnowledge` view (their FR-7 reframe). Collection here, accumulation there — no
  duplicate machinery.
- **FR-H5 — Evidence vs declaration precedence.** Where derived evidence and a human declaration
  conflict: **evidence wins for field-sets** (the schema is ground truth); **declaration wins for
  choices not derivable from evidence** (framework, naming for new code). **Module paths are
  per-aspect** (fixing the brownfield conflict — `project_knowledge` derives module-path
  positives *and* negatives from code, REQ-522/D2, so in brownfield they ARE evidence):
  evidence wins for *existing* module paths; declaration wins for *new-subsystem* paths; a
  declaration contradicting derived paths for existing modules (e.g. declared
  `module_paths.tables: app.models` against a codebase using `app.tables`) is a **flagged
  conflict with evidence precedence**, never a silent declaration win. Conflicts are flagged at
  **preflight (startd8 `plan_ingestion`)** — the owning surface — never silently resolved.

## 4. Acceptance (Group H)

- A convention-strict view task routed to micro-prime has framework/module-path authority
  **visible in its prompt artifact**, or the task is re-routed — inspectable per run (FR-H2).
- A greenfield run with a declared manifest produces zero RUN-028-class inventions
  (Flask/`app.models`/SQLAlchemy) in a re-run of the **pinned RUN-028 workload** (cite the
  archived run id + seed path when pinning), scored with the **structural scorer** from the
  adherence-validation methodology — never denylist scoring (it misleads both ways); two
  independent runs of the test must agree.
- The greenfield acceptance MUST also pass with the hardcoded fallback **disabled**
  (`render_convention_guidance()` replaced by an empty string) — proving the FR-H1
  declared-manifest path, not the Python house-style fallback, does the work.
- The seed's provenance shows the convention manifest path + hash + status; the FR-X1 report
  shows `conventions: declared` (or `absent → invention risk` with the affected tiers named).
- A deliberate evidence-vs-declaration conflict (declared field absent from the schema) is
  flagged, not silently resolved (FR-H5).
