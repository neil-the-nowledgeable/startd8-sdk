## Technology conventions

▷ Key-lines first (free-text prose above them is tolerated; the stack table ends the key block).
▷ `Language` is **required** — without it there is no valid candidate and nothing round-trips.
▷ `Field authorship` points at the human-only inputs file (fields the AI must never write).
▷ `Provenance default` is `authored` once the architect validates, else `templated` (prototype starters).

- Language: <python>
- Field authorship: <docs/kickoff/inputs/human_inputs.yaml>
- Provenance default: <authored | templated>

▷ The stack — one row per layer. `Layer` normalizes to a `conventions.yaml` key (`data layer`→`data_layer`);
▷ `Choice` is the value. Name the choice AND what it is *not* where there's a common confusion.

| Layer | Choice | Not |
|-------|--------|-----|
| framework | <fastapi> | <not Flask/Django> |
| data layer | <sqlmodel> | <not bare SQLAlchemy> |
| templates | <jinja2> | |
| frontend | <htmx> | <not React/SPA> |
| database | <sqlite> | <dev; Postgres in prod> |

### Naming

▷ `classes`→`classes`, `metric prefix`→`metric_prefix`. An unrecognized aspect is kept (spaces→underscore),
▷ never dropped.

| Aspect | Style |
|--------|-------|

| files | <snake_case> |
| classes | <PascalCase> |

### Data-model conventions

▷ The non-derivable modeling choices generation would otherwise INVENT — declared **once** here, never
▷ per entity (KICKOFF_REQUIREMENTS FR-F6). Each is a `- Key: value` line. Out-of-vocab values are
▷ **flagged** (`not_extracted`), never guessed. Valid values:
▷   money: cents | float        (cents = integer minor units → exact sums)
▷   datetime: utc | local        (name the timezone policy)
▷   recurrence: structured | rrule | none
▷   references: fk-only | loose-allowed   (may an entity hold a loose polymorphic `references` id?)
▷   weekday: iso | us             (optional)

- money: <cents>
- datetime: <utc>
- recurrence: <none>
- references: <fk-only>

#### Computed fields

▷ Fields DERIVED at runtime (no form input). Bullets, verbatim — e.g. `Order.total`.

- <Entity.field — what it is computed from>

#### Deferred

▷ What is explicitly OUT of v1 (mirror requirements Non-goals). Bullets, verbatim.

- <capability deliberately deferred>

### Architecture invariants

▷ The 2–3 things a new generator tier MUST know — verbatim bullets (contract-first / monolith-vs-services /
▷ determinism preference). These ride through as `architecture_notes`.

- <contract-first: schema.prisma is the single source of truth; models derive from it>
- <modular monolith now; service-extraction-ready boundaries>
- <prefer deterministic codegen; LLM only at the integration seam>

## Reference (prose — ignored by extraction)

| Prose | → `conventions.yaml` | Rule |
|---|---|---|
| `- Language:` | `language` | **required** key-line; absent ⇒ nothing extracts |
| `- Field authorship:` / `- Provenance default:` | `field_authorship` / `provenance_default` | scalar key-lines |
| `\| Layer \| Choice \|` table | `stack.<layer>` | Layer→key (`data layer`→`data_layer`), Choice→value |
| `### Module layout` `\| Role \| Path \|` | `module_paths.<role>` | Role→key, Path→value |
| `### Naming` `\| Aspect \| Style \|` | `naming.<key>` | aspect synonyms → canonical keys |
| `### Data-model conventions` `- key: value` | `data_model.<key>` | enum-checked; bad value/unknown key → flagged |
| `#### Computed fields` / `#### Deferred` bullets | `data_model.computed_fields[]` / `data_model.deferred[]` | verbatim bullet lists |
| `### Architecture invariants` bullets | `architecture_notes[]` | verbatim bullets |

A data-model value outside its enum, or an unknown data-model key, is recorded `not_extracted` in the
`kickoff check` report (never guessed, contract §3). `Language` is the only hard-required field.
