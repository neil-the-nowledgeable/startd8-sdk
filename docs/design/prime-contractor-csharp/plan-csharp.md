## Overview

Implement the single C# microservice from Google's Online Boutique microservices demo. This is a validation exercise for the capability delivery pipeline: the reference implementation provides an unambiguous target for structural comparison.

**Objectives:** Implement the C# cartservice from Google's Online Boutique microservices demo as a structurally equivalent, diffable reproduction of the reference implementation.

### Goals

- Produce one functional C# microservice: cartservice
- ASP.NET Core gRPC service implementing the CartService proto contract (AddItem, GetCart, EmptyCart)
- Three cart storage backends: Redis (IDistributedCache), Spanner, AlloyDB (Npgsql)
- gRPC health checking that delegates to cart store `Ping()`
- Unit tests using xUnit with ASP.NET Core TestHost
- Containerized with multi-stage Dockerfile using `dotnet/sdk` builder and `dotnet/runtime-deps` runtime (self-contained, trimmed)
- Generated code is structurally comparable to the reference implementation

### Non-Goals

- Kubernetes manifests, Helm charts, Kustomize overlays (infrastructure-as-code)
- Services in other languages (Python, Go, Node.js, Java)
- CI/CD pipelines
- `Dockerfile.debug` (development-only Dockerfile)
- `.dockerignore` files
- Apache 2.0 license headers (copyright boilerplate)

### Delivery Mechanism

Every feature produces **valid C# files and project configuration** written to the target project root. Success is measured by structural equivalence to the reference implementation: same files, same interfaces, same patterns.

### F-003: Cart Store Interface + Redis Implementation

**Implementation contract:**

`cartstore/ICartStore.cs` (26 lines):
- `namespace cartservice.cartstore`
- Import: `System.Threading.Tasks`
- `public interface ICartStore`:
  - `Task AddItemAsync(string userId, string productId, int quantity)`
  - `Task EmptyCartAsync(string userId)`
  - `Task<Hipstershop.Cart> GetCartAsync(string userId)`
  - `bool Ping()`

`cartstore/RedisCartStore.cs` (118 lines):
- `namespace cartservice.cartstore`
- Imports: `System`, `System.Linq`, `System.Threading.Tasks`, `Grpc.Core`, `Microsoft.Extensions.Caching.Distributed`, `Google.Protobuf`
- `public class RedisCartStore : ICartStore`:
  - `private readonly IDistributedCache _cache` (injected)
  - Constructor: `public RedisCartStore(IDistributedCache cache)`
  - `AddItemAsync`:
    - `Console.WriteLine($"AddItemAsync called with userId={userId}, productId={productId}, quantity={quantity}")`
    - `var value = await _cache.GetAsync(userId)`
    - If null: creates new Cart with UserId and CartItem
    - If exists: `Hipstershop.Cart.Parser.ParseFrom(value)`, finds `SingleOrDefault(i => i.ProductId == productId)`, adds or increments quantity
    - `await _cache.SetAsync(userId, cart.ToByteArray())`
    - Catch → `RpcException(StatusCode.FailedPrecondition, "Can't access cart storage. {ex}")`
  - `EmptyCartAsync`:
    - Creates new empty Cart, saves via `SetAsync` (overwrites, doesn't delete)
    - Same exception pattern
  - `GetCartAsync`:
    - Gets from cache, parses or returns empty Cart
    - Same exception pattern
  - `Ping()`: returns `true` (try/catch returning false)
